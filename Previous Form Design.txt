═══════════════════════════════════════════════════════════════════════════════
                            FORM OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

The Complete Lease Accounting Form is organized into 6 main tabs at the top of 
the form. Users navigate between tabs by clicking the tab buttons. Fields marked 
with a red asterisk (*) are MANDATORY and must be filled before saving.

KEY FEATURES:
  • AI-Assisted Data Entry: Upload PDF lease documents to auto-populate fields
  • Conditional Logic: Some fields appear/disappear based on toggle switches
  • Dynamic Fields: Add multiple entries for rentals, ARO revisions, security 
    deposits, etc.
  • Tab-based Navigation: Organized into logical sections

═══════════════════════════════════════════════════════════════════════════════
                        TAB 2: DATES & TERMS
═══════════════════════════════════════════════════════════════════════════════

This tab contains all date-related fields and lease term parameters.

───────────────────────────────────────────────────────────────────────────────
                              Mandatory Fields
───────────────────────────────────────────────────────────────────────────────

  Lease Start Date (*)
    Purpose:       Date when lease commences (lease inception date)
    Display:       Date picker (calendar widget)
    Accounting     Critical for calculating lease term and present value
      Note:        
    IFRS/Ind AS    ASC 842 / Ind AS 116
      Reference:   
    Example:       January 1, 2025

  First Payment Date (*)
    Purpose:       Date of first lease payment
    Display:       Date picker
    Accounting     May differ from lease start date; affects payment schedule
      Note:        
    Example:       Lease starts Jan 1, first payment due March 1

  Lease End Date (*)
    Purpose:       Date when lease term expires
    Display:       Date picker
    Accounting     Used to calculate lease term and remaining lease liability
      Note:        

  Rent Accrual Day (*)
    Purpose:       Day of month when rent is recognized in books
    Display:       Number input (1-31)
    Accounting     For accrual accounting; typically matches payment date
      Note:        
    Example:       5 (rent accrues on 5th of each month)

  Rental Frequency (Months) (*)
    Purpose:       Frequency of rental payments
    Display:       Dropdown selection
    Options:       Monthly (1), Quarterly (3), Semi-annually (6), Annually (12)
    Accounting     Determines payment schedule and discounting frequency
      Note:        

  Day of Month (*)
    Purpose:       Specific day of month when payments are due
    Display:       Dropdown selection
    Options:       1-31, or "Last Day" for month-end payments
    Accounting     Combined with frequency to generate payment schedule
      Note:        
    Example:       Day 5 = payment due on 5th of each month

───────────────────────────────────────────────────────────────────────────────
                              Optional Fields
───────────────────────────────────────────────────────────────────────────────

  Agreement Date
    Purpose:       Date when lease agreement was signed
    Display:       Date picker
    Accounting     May differ from lease start date
      Note:        

  Termination Date
    Purpose:       Date when lease was terminated (if applicable)
    Display:       Date picker
    Accounting     For terminated/modified leases
      Note:        

  Termination Penalty
    Purpose:       Amount payable on early termination
    Display:       Number input (decimal allowed)
    Accounting     Liability recognition requirement
      Note:        

  Tenure (Months)
    Purpose:       Total lease term in months
    Display:       Number input (decimal allowed)
    Accounting     Automatically calculated from start and end dates if not 
      Note:        provided
    Example:       60 months (5 years)

                            TAB 3: RENTALS
═══════════════════════════════════════════════════════════════════════════════

This tab contains all rental payment information with conditional logic.

───────────────────────────────────────────────────────────────────────────────
                            Auto Rentals Section
───────────────────────────────────────────────────────────────────────────────

Toggle Switch: Auto Rentals (On/Off)

  Display:        Toggle switch with label showing "Yes" or "No"
  How it works:   When turned ON, a rental amount field appears below
  Accounting      
    Note:         
    • ON (Yes):   System calculates rental payments automatically based on 
                  frequency
    • OFF (No):   Manual rental schedule must be entered

When Auto Rentals is ON:

  Rental Amount (*)
    Purpose:       Fixed monthly/periodic rental payment amount
    Display:       Number input (appears when Auto Rentals is ON)
    Accounting     Base rental for payment schedule calculation
      Note:        
    Example:       $10,000 per month

───────────────────────────────────────────────────────────────────────────────
                          Static Escalation Section
───────────────────────────────────────────────────────────────────────────────

Toggle Switch: Static Escalation (On/Off)

  Display:        Toggle switch with label showing "Yes" or "No"
  How it works:   When turned ON, escalation fields appear below
  Accounting      For leases with scheduled rent increases
    Note:        

When Static Escalation is ON:

  Escalation Start Date
    Purpose:       Date when escalation begins
    Display:       Date picker (appears when Escalation is ON)
    Accounting     First date of escalated rent
      Note:        

  Escalation Percentage (%)
    Purpose:       Percentage increase in rental amount
    Display:       Number input (decimal allowed)
    Accounting     Applied at escalation frequency intervals
      Note:        
    Example:       5% annual increase

  Escalation Frequency (Months)
    Purpose:       How often escalation applies
    Display:       Number input
    Accounting     Combined with percentage to calculate escalated rents
      Note:        
    Example:       12 months = annual escalation

  Index/Rate Table
    Purpose:       Variable rate index reference (if applicable)
    Display:       Number input
    Accounting     For CPI-linked or variable rate leases
      Note:        

───────────────────────────────────────────────────────────────────────────────
                         Manual Adjustments Section
───────────────────────────────────────────────────────────────────────────────

Toggle Switch: Manual Adjustments (On/Off)

  Display:        Toggle switch with label showing "Yes" or "No"
  How it works:   When turned ON, manual rental schedule section appears
  Accounting      Used when rental payments don't follow a regular pattern
    Note:        

When Manual Adjustments is ON:

  Manual Rental Schedule
    Purpose:       Define specific rental amounts for specific dates
    Display:       Dynamic list with "Add Rental" button
    How it works:  
      1. Click "+ Add Rental" button
      2. Enter Rental Amount
      3. Enter Rental Date
      4. Repeat for each payment
    Accounting     For irregular payment schedules (e.g., free rent period, 
      Note:        stepped rents)
    Example:       Month 1-3: $0 (free rent), Month 4-12: $12,000, Month 13+: 
                  $15,000

This tab contains discount rates, ARO, security deposits, and other financial

───────────────────────────────────────────────────────────────────────────────
                          Basic Financial Fields
───────────────────────────────────────────────────────────────────────────────

  Borrowing Rate (%) (*)
    Purpose:       Discount rate used for present value calculations 
                   (incremental borrowing rate)
    Display:       Number input (decimal allowed)
    Accounting     Critical for lease liability and ROU asset measurement
      Note:        
    IFRS/Ind AS    ASC 842 / Ind AS 116 requires IBR or lease-specific rate
      Reference:   
    Example:       5.5% per annum

  Currency
    Purpose:       Transaction currency
    Display:       Dropdown selection
    Options:       USD, EUR, INR, GBP
    Accounting     For multi-currency reporting
      Note:        

  Fair Value of ROU Asset
    Purpose:       Fair value of right-of-use asset (if available)
    Display:       Number input (decimal allowed)
    Accounting     Alternative measurement if available (rare)
      Note:        

  Compound Months
    Purpose:       Compounding frequency for discount rate
    Display:       Number input
    Accounting     Typically matches rental frequency (monthly = 1, quarterly = 
      Note:        3, etc.)
    Default:       Auto-derived from rental frequency if not specified

───────────────────────────────────────────────────────────────────────────────
          Asset Retirement Obligation (ARO) Section
───────────────────────────────────────────────────────────────────────────────

Toggle Switch: Asset Retirement Obligation (ARO) (On/Off)

  Display:        Toggle switch with label showing "Yes" or "No"
  Accounting      For leases where lessee must restore asset at end of lease
    Note:        

When ARO is ON:

  ARO Initial Amount
    Purpose:       Present value of estimated restoration costs
    Display:       Number input (appears when ARO is ON)
    Accounting     Liability recognized at lease inception (ASC 410 / Ind AS 
      Note:        37)

  ARO Interest Table
    Purpose:       Interest rate for ARO accretion
    Display:       Number input
    Accounting     Typically risk-free rate or borrowing rate
      Note:        

  ARO Revisions (up to 8)
    Purpose:       Subsequent adjustments to ARO estimate
    Display:       Dynamic list with "Add ARO Revision" button
    How it works:  
      1. Click "+ Add ARO Revision" button
      2. Enter Revision Amount
      3. Enter Revision Date
    Accounting     Revisions adjust ARO liability and asset cost 
      Note:        (prospectively)

───────────────────────────────────────────────────────────────────────────────
                        Security Deposits Section
───────────────────────────────────────────────────────────────────────────────

Toggle Switch: Security Deposits (On/Off)

  Display:        Toggle switch with label showing "Yes" or "No"
  Accounting      Refundable deposits held by lessor
    Note:        

When Security Deposits is ON:

  Security Deposit Amount (Gross)
    Purpose:       Total deposit amount paid to lessor
    Display:       Number input (appears when Security Deposits is ON)

  Security Discount Rate (%)

    Display:       Number input

      Note:

      1. Click "+ Add Security Increase" button

      Note:

═══════════════════════════════════════════════════════════════════════════════
                       FIELD DISPLAY PATTERNS
═══════════════════════════════════════════════════════════════════════════════

1. STANDARD TEXT INPUT
   Appearance:    Single-line rectangular box
   Use:            For text, numbers, codes
   Example:        Asset Name, Counterparty, Cost Center

2. TEXT AREA
   Appearance:     Multi-line box (3 rows typically)
   Use:            For longer descriptions
   Example:        Description field

3. DATE PICKER
   Appearance:     Text input with calendar icon; clicking opens calendar widget
   Use:            For all date fields
   Example:        Lease Start Date, First Payment Date

4. DROPDOWN SELECTION
   Appearance:     Clickable box showing current selection; clicking opens list
   Use:            For fields with predefined options
   Example:        Asset Class, Currency, Rental Frequency

5. NUMBER INPUT
   Appearance:     Text input box (typically right-aligned for numbers)
   Use:            For amounts, rates, percentages
   Example:        Rental Amount, Borrowing Rate, Escalation Percentage

6. TOGGLE SWITCH
   Appearance:     Slider switch (like on/off button)
   Use:            For Yes/No decisions that show/hide other fields
   Behavior:      
     • Left (OFF) = No (gray)
     • Right (ON)  = Yes (green)
     • Label text shows current status: "Yes" or "No"
   Example:        Auto Rentals, Escalation, ARO, Security Deposits

7. DYNAMIC LISTS
   Appearance:     List of input rows with "+ Add" button
   Use:            For multiple entries (rentals, ARO revisions, etc.)
   Behavior:      
     • Click "+ Add [Item]" to add new row
     • Each row has input fields
     • Each row has "Remove" button to delete
   Example:        Manual Rental Schedule, ARO Revisions, Security Increases

                          TIPS FOR CA USERS
═══════════════════════════════════════════════════════════════════════════════

1. Start with General Information: Complete asset identification first

2. Use AI Extraction: Upload PDF first to auto-populate many fields

3. Verify All Dates: Check that dates are logically consistent

4. Check Toggle Switches: Ensure correct toggles are ON/OFF for your lease type

5. Review Calculations: System calculates PV factors and schedules 
   automatically

6. Document Upload: Upload original lease agreement for audit trail

7. Save Frequently: Use "Save Draft" to avoid data loss
